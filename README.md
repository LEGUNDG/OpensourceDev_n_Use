# 🎭 CULTURE LOG
오픈소스 개발과 활용 (13–14주차 조별 프로젝트)
CULTURE LOG는 영화·연극·뮤지컬 관람 경험을 기록하고 공유하는 정적 웹 아카이브이다.
소비로 끝나는 문화생활을 글과 이미지로 정리하여, 개인의 문화적 경험을 구조화된 기록으로 남기는 것을 목표로 한다.

## 📌 프로젝트 주제
주제: 문화생활 기록 아카이브 웹사이트 제작
소주제: 영화 / 연극 / 뮤지컬
팀원별로 하나의 장르를 맡아 콘텐츠 조사, 작성, 웹 페이지 구현을 수행하였다.

## 🎯 프로젝트 목표
- 오픈소스 기반 정적 웹사이트 구조 이해
- GitHub 및 GitHub Pages를 활용한 협업·배포 경험
- 역할 분담을 통한 협업 개발 프로세스 학습
- 문화 콘텐츠를 정보 구조로 표현하는 능력 강화

## 🛠 사용 기술 및 개발 환경
- HTML5 / CSS3
- Git & GitHub
- GitHub Pages
- Visual Studio Code
서버 및 프레임워크를 사용하지 않고, 오픈소스 도구만을 활용한 정적 웹 개발을 목표로 하였다.

## 🖥 주요 기능
1. 메인 페이지
 - 사이트 소개
 - 영화 / 연극 / 뮤지컬 카테고리 제공
 - 전체 콘텐츠 목록 탐색
2. 상세 페이지
 - 작품 이미지
 - 기본 정보 및 제작 배경
 - 줄거리 요약
 - 주요 인물 및 OST(또는 인상 장면)
 - 개인 감상평
3. 배포 방식
   : GitHub Pages를 활용한 정적 웹사이트 배포

## 📌 WBS (Work Breakdown Structure)
1. 프로젝트 기획
 - 주제 선정
 - 사이트 콘셉트 정의
 - 콘텐츠 카테고리 확정
 - 팀원 역할 분담
2. 설계 단계
 - 전체 사이트 구조 설계
 - 메인/상세 페이지 구성 정의
 - 폴더 및 파일 구조 설계
3. 콘텐츠 제작
 - 영화 콘텐츠 조사 및 작성 (최운혁)
 - 연극 콘텐츠 조사 및 작성 (이정민)
 - 뮤지컬 콘텐츠 조사 및 작성 (최찬영)
4. 웹 구현
 - 메인 페이지 HTML 구현
 - 상세 페이지 HTML 구현
 - CSS 스타일 적용 및 이미지 정리
5. 통합 및 테스트
 - 콘텐츠 통합
 - 링크 및 경로 오류 점검
 - 브라우저 테스트
6. 배포 및 문서화
 - GitHub 저장소 정리
 - GitHub Pages 배포
 - README 문서 작성
 - 최종 제출 자료 정리
